## 📊 Detailed Data Flow: Creating an Order

**Step-by-Step Process:**

1. **Client Sends Request**
   ```json
   POST /api/orders
   {
     "items": [
       {"price": 150, "quantity": 2},
       {"price": 300, "quantity": 1}
     ],
     "customerId": "customer123"
   }
   ```

2. **[`app.js:8`](app.js:8)** - Middleware Processing
   - Parses JSON body
   - Routes to `/api/orders` endpoint

3. **[`routes/orders.js:36`](routes/orders.js:36)** - Request Validation
   - Validates items array exists and is non-empty
   - Extracts `items` and `customerId`

4. **[`services/pricing.js:11`](services/pricing.js:11)** - Price Calculation
   ```javascript
   // Calculate subtotal
   calculateTotal(items) → (150 × 2) + (300 × 1) = 600
   
   // Calculate discount
   calculateDiscount(600) → 600 ≥ 500, so 10% = 60
   
   // Final price
   finalPrice = 600 - 60 = 540
   ```

5. **[`routes/orders.js:51`](routes/orders.js:51)** - Order Creation
   - Generates unique order ID
   - Creates order object with pricing details
   - Stores in in-memory array

6. **Response to Client**
   ```json
   HTTP 201 Created
   {
     "success": true,
     "data": {
       "id": 1,
       "customerId": "customer123",
       "totalPrice": 600,
       "discount": 60,
       "finalPrice": 540,
       "status": "pending"
     }
   }
   ```

## 💰 Pricing Service Logic

The [`pricing.js`](services/pricing.js) module implements tiered discounts:

| Total Price | Discount | Example |
|-------------|----------|---------|
| ≥ $1000 | 15% | $1200 → $180 off |
| ≥ $500 | 10% | $600 → $60 off |
| ≥ $100 | 5% | $150 → $7.50 off |
| < $100 | 0% | $50 → No discount |

**Key Functions:**
- [`calculateTotal(items)`](services/pricing.js:11) - Sums item prices × quantities
- [`calculateDiscount(totalPrice)`](services/pricing.js:28) - Applies tiered discounts
- [`calculateTax(price, taxRate)`](services/pricing.js:49) - Calculates tax (default 8%)
- [`calculateFinalPrice(items, taxRate)`](services/pricing.js:59) - Complete pricing breakdown

## 🏛️ Architecture Principles

**3-Tier Design:**
- **Presentation Layer** ([`routes/orders.js`](routes/orders.js)) - HTTP handling, validation
- **Business Logic Layer** ([`services/pricing.js`](services/pricing.js)) - Pure calculations
- **Data Layer** - In-memory storage (demo purposes)

**Key Features:**
- RESTful API design with proper HTTP methods
- Centralized error handling in [`app.js:26`](app.js:26)
- Modular, testable service functions
- Clean separation of concerns
